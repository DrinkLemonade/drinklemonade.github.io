Namespace Gameplay

Classes

ButtonEffects
ButtonEffects is used by a variety of button-like UI elements, allowing them to grow when clicked, or shake to indicate an incorrect player interaction.

CategoryPanel
The CategoryPanel class is a component script used by CategoryPanel prefabs, but currently has no use. Future uses could include special visual effects when a category is found (creating sparkles linked to the category's color, for instance).

ConfirmButton
Calls GameManager.i.currentSession.EvaluateGuess(); when clicked.

FadeMixerGroup
Creates a functional audio fade-in/fade-out effect for the game's AudioMixer. Original code by johnleonardfrench.

GameEndPanel
Shows success or failure text when the game ends. Includes a "back to main menu" function called by a button child of the associated GameObject.
GameManager
The GameManager singleton covers transition between game states, handling events such as changes in music when the game switches to the Start state, or creating and holding a GameSession object.

GameMenuLogic
Logic for the menu UI used in the Gameplay scene. Allows quitting the game and restarting the puzzle scene.

GameSession
A GameSession is an ongoing game of CMC. The class's constructor takes in information regarding which words and categories are in play. This class is responsible for handling game logic: evaluating the player's word guesses, rewarding or punishing the player when a guess is correct or incorrect, keeping track of previously-attempted guesses, calling the hint UI when the player is only missing one word or repeating a previous guess, and finally calling the victory/defeat UI when conditions are met.

GameSettings
The GameSettings class is unfinished and cannot be accessed in-game. This class is intended to provide UI functionality for the game's "Settings" panel when implemented.

InfoBanner
The InfoBanner class is used by the InfoBanner GameObjects which show themselves when the player makes a nearly-correct guess or repeats a previously-attempted guess.

LevelSelectButton
Used by the puzzle selection buttons in the main menu scene.

LivesTracker
Can be called to update the UI GameObject which tracks the player's remaining attempts.

MainMenu
MainMenu is responsible for transitions between sub-menus (the start screen and the puzzle list, for instance). It parses retrieved JSON level data and transforms it into a list of GameLevel structs, which inform the button grid that makes up the puzzle list sub-menu.

MusicSettingsChanger
The MusicSettingsChanger class cannot be accessed in-game. This class is intended to update the game's AudioMixer values when GameSettings is implemented.

PlayFabLogin
When the game enters its main menu, this classes attempts to establish a connection to the PlayFab server which holds the game's level data. If successful, this level data is downloaded in JSON format, which is then sent to the MainMenu class in order to be parsed and processed.

RepeatingTiledBackground
Tweens an Image in a seamless repeating loop. Used by the Backgrounds prefab.

SettingsPanelWritePrefs
The SettingsPanelWritePrefs class cannot be accessed in-game. This class is intended to write to a file to save the player's chosen settings after GameSettings is implemented.

ShuffleExtension
A static class that can be used to shuffle a list like a deck of cards. Used by UIManager to shuffle the order of words on the screen at the beginning of a game.

SoundManager
A singleton which plays or loops AudioClips and calls FadeMixerGroup for smooth fade-ins/fade-outs.

TitleScreenWord
Governs the animation of title screen logo words.

UIManager
UIManager handles the main gameplay scene's UI logic. It obtains the current puzzle information from GameManager's ongoing GameSession and arranges the puzzle's various words into a grid. Various methods are responsible for calling the coroutines used to animate the game's UI, such as GridAnimation. Logic also exists to check the player's interaction with the UI, for instance allowing the submission of a guess attempt only if the attempts contains no more and no less than 4 words. Methods to enable the game's victory and defeat UI panel also exist for GameSession to call when the conditions are met.

WordToggle
Used by the WordToggle GameObjects which are the player's main mode of interaction with the core gameplay. This class handles the toggle's logic and animation.

WordToggleAnchor
A dummy class used in place of a tag system in order to find each WordToggleAnchor GameObject and link it to its associated WordToggle.

GameLevel
Keeps information relative to a puzzle: unique ID and categories used.
